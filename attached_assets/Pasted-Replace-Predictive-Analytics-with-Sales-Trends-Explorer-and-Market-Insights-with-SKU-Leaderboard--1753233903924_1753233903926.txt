🟠 SECTION 1: Replace Predictive Analytics with Sales Trends Explorer
🔁 Purpose
Substitute the AI forecasting with a time-series visual of past performance using real sales data.

Page Changes
Rename the Predictive Analytics tab to Sales Trends Explorer.

Replace chart area with a line or bar chart (use Recharts, Chart.js, or Victory.js).

Inputs / Filters (Reuse existing logic from Sale History)
Date Range (default last 30 days)

SKU filter

Category selector

Data Displayed
Line Chart (X-axis = date)

Revenue over time

Units sold

Optional toggle: profit, avg order value

Chart Logic
Use toggles or checkboxes:

📊 Revenue

📦 Units Sold

🟠 SECTION 2: Replace Market Insights with SKU Leaderboard
🔁 Purpose
Instead of vague AI suggestions, give users a ranked leaderboard of their best- and worst-performing products.

✅ Frontend Tasks
Page Changes
Rename Market Insights tab to SKU Leaderboard

Replace "View Market Analysis" card with a sortable table or card grid view.

Display Metrics (for each SKU):
SKU	Product	Units Sold	Revenue	Profit	Avg Order Value	Margin (%)

Add sort buttons for:

Most sold

Highest revenue

Highest profit margin

Lowest performer

Optional badges:

🥇 “Top Performer”

🚨 “Needs Review” (if low profit or negative)

✅ Backend Tasks
API Route: GET /api/analytics/sku-leaderboard

Backend Logic:

Aggregate sales_history by SKU

Join with inventory for cost and price

Calculate margin = (profit / revenue) × 100

Return sorted list of SKUs with metrics

✅ Reusability Across Pages
You can reuse leaderboard data for:

Dashboard Trends

Goal Progress (e.g., “Top SKU target met”)

Product Workspace insights
